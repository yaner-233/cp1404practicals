## Key Lessons Learned About "Clean Code"  
- **Single Responsibility Principle**: Every function, class, or module should handle one specific task. For example, in the project management program, "loading data," "saving data," and "displaying data" are separated into distinct functionsâ€”this makes code easier to debug and extend.  
- **Meaningful Naming & Comments**: Variable, function, and class names should directly reflect their purpose. Comments should explain "why" rather than "what"; clear naming reduces reliance on comments, making code self-documenting.  
- **Consistency & Simplicity**: Maintain consistent coding style (indentation, naming conventions, blank lines) and avoid redundant logic. For instance, using list comprehensions to simplify loops or flag variables to replace repetitive condition checks improves readability and reduces errors.
